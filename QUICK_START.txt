🚀 QUICK START - ADMIN SYSTEM SETUP

═══════════════════════════════════════════════════════════════════

📖 READ THIS FIRST: MASTER_CHECKLIST.md

Then follow these 9 phases:

═══════════════════════════════════════════════════════════════════

PHASE 1: Préparation (5 min)
  ✓ Read MASTER_CHECKLIST.md
  ✓ Read ADMIN_STRUCTURE.md

PHASE 2: Setup Supabase (15 min)
  Read: SUPABASE_SETUP.md
  1. Create account: https://supabase.com
  2. Create project
  3. Run db-setup.sql (SQL Editor)
  4. Run db-functions.sql (SQL Editor)
  5. Copy Project URL & Service Role Key

PHASE 3: Setup Resend (5 min)
  Read: RESEND_SETUP.md
  1. Create account: https://resend.com
  2. Create API Key
  3. Copy API Key

PHASE 4: Configuration Local (10 min)
  Create .env.local with:
  - NEXT_PUBLIC_SUPABASE_URL
  - SUPABASE_SERVICE_ROLE_KEY
  - RESEND_API_KEY
  - ADMIN_EMAIL
  - NEXT_PUBLIC_ADMIN_PASSWORD
  - NEXT_PUBLIC_URL=http://localhost:3000
  - NODE_ENV=development

PHASE 5: Installation (5 min)
  $ yarn install

PHASE 6: Testing Local (30 min)
  Read: LOCAL_TESTING.md
  $ yarn dev
  - Test admin login
  - Test contact form
  - Test newsletter
  - Verify emails received

PHASE 7: Intégrer Formulaires (10 min)
  - Add ContactFormNew to contact page
  - Add NewsletterForm to site (footer, etc)
  - Test all forms work

PHASE 8: Push to GitHub (5 min)
  $ git add .
  $ git commit -m "feat: Add admin system"
  $ git push origin main

PHASE 9: Deploy Vercel (20 min)
  Read: VERCEL_DEPLOYMENT.md
  1. Create account: https://vercel.com
  2. Import project from GitHub
  3. Add 7 environment variables
  4. Deploy
  5. Test on production URL

═══════════════════════════════════════════════════════════════════
